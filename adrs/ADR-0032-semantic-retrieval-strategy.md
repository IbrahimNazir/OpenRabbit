# ADR-0032: Semantic Retrieval Strategy

**Status:** Accepted
**Date:** 2025-02-27
**Phase:** Phase 3 — RAG/Context Engine

---

## Context

The review pipeline needs two types of context from the vector index:
1. Semantically related code (for understanding what the changed code interacts with)
2. Direct call sites of changed functions (for breaking-change detection in Stage 3)

These have different retrieval requirements.

## Decision

Two retrieval modes in `ContextRetriever`:

### `find_relevant_context` — Vector Similarity Search
- Embed the NL query (generated by `_describe_change`)
- Qdrant nearest-neighbour search with `must=[repo_id match]` + `must_not=[exclude changed files]` (only added when `exclude_files` is non-empty — Qdrant rejects empty `MatchAny`)
- Score threshold: `>= 0.75` (below this, results are not semantically related)
- `top_k = 5` results returned

### `find_callers` — Text Payload Index Keyword Search
- Qdrant scroll with `MatchText(function_name)` on the `content` field
- Requires the `content` field text index created by `ensure_collection()`
- Returns up to 10 results, sorted by `(file_path, start_line)`
- Falls back to empty list if Qdrant text index is unavailable

**Token budget:** Total injected RAG context trimmed to 4000 tokens using tiktoken `cl100k_base`. Chunks sorted by score descending, greedily included until budget is reached.

**Context injection:** Prepended to `full_file_context` in Stage 2 (`PROMPT_BUG_DETECTION`). Stage 3 receives caller chunks directly as structured `call_sites` dicts.

## Consequences

- Score threshold 0.75 filters low-quality results; if nothing scores that high (e.g., repo not fully indexed), returns empty list → Phase 2 behavior preserved
- Empty `exclude_files` guard prevents Qdrant API errors
- Lazy tiktoken import with try/except — graceful if tiktoken unavailable

## Alternatives Considered

| Option | Reason Rejected |
|--------|----------------|
| Top-k without threshold | Injects irrelevant code, polluting prompts |
| BM25 for all retrieval | No vector model needed but misses semantic matches (same concept, different naming) |
| Full-text search only | Misses semantically related code that doesn't share identifiers |
